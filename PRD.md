## 1. 产品背景

### 1.1 现状与痛点

财务人员在日常工作中需要频繁查询和分析序时账（General Ledger）数据，当前主流工作方式存在以下问题：

| 痛点 | 具体表现 |
|------|---------|
| **操作繁琐** | 每次查询需打开 Excel → 筛选 → 透视表 → 写公式，一个简单问题耗时 5-15 分钟 |
| **技能门槛高** | 需要掌握 Excel 高级功能（VLOOKUP、透视表、公式嵌套）或 SQL 技能 |
| **重复劳动** | 类似的查询需求反复执行（"某某科目的本月发生额"、"某部门的费用汇总"），每次手动操作 |
| **数据验证困难** | 需要人工核对序时账与科目余额表的一致性，缺乏自动化验证手段 |
| **安全顾虑** | 财务数据敏感，企业不允许上传至公有云服务 |
| **工具割裂** | 分析过程中需要在 Excel、数据库客户端、计算器等多个工具之间切换 |

### 1.2 机会

大语言模型（LLM）在代码生成领域已展现足够能力，可以将自然语言查询直接翻译为 Pandas 代码，在本地安全执行。DeepSeek API 提供高性价比的中文场景支持。

### 1.3 产品定位

> **本地部署、AI 驱动、自然语言交互的财务数据查询分析工具**  
> 让非技术用户通过说中文就能查询财务数据，无需编写任何代码。

### 4.2 异常流程

**上传失败** → 提示不支持的文件类型 / 编码检测失败 → 建议用户另存为 UTF-8 CSV

**映射字段缺失** → 标红必填字段，阻止进入下一步 → 提示"X 是必填字段"

**AI 代码生成失败** → 返回具体错误（API Key 无效 / 网络超时 / 模型返回异常）

**代码执行失败** → 自动尝试二次执行（数据清洗增强）→ 三次执行（极端清洗）→ 显示增强错误提示

## 5. 技术需求

### 5.1 架构要求

- **纯本地部署**：所有数据在用户机器上处理，不上传任何第三方服务器（除 DeepSeek API 调用）
- **无数据库依赖**：使用 Flask-Session 文件存储，无需安装 MySQL/PostgreSQL
- **单文件启动**：`python app.py` 即可运行

### 5.2 安全要求

- **代码沙箱**：AI 生成的 Pandas 代码必须经过 AST 静态检查 + 受限执行环境 + 超时控制三层防护
- **会话隔离**：每个浏览器会话独立，session 文件无共享
- **API Key 安全**：仅存于服务端 session 文件，不写日志、不入库

### 5.3 兼容性要求

- **操作系统**：macOS / Windows / Linux
- **Python**：3.9+
- **浏览器**：Chrome 90+ / Edge 90+ / Safari 14+ / Firefox 90+
- **文件格式**：xlsx / csv

### 5.4 性能要求

| 指标 | 目标 |
|------|------|
| 文件上传解析 | < 3s（100MB 以内） |
| 字段映射保存 | < 1s |
| AI 代码生成 | < 10s（取决于 DeepSeek API 响应） |
| 代码执行 | < 30s（默认超时） |
| 完整性测试 | < 10s（100 万行以内） |
| Excel 导出 | < 5s（10 万行以内） |

## 6. 非功能需求

### 6.1 易用性
- 5 步引导式流程，步骤进度条清晰显示当前位置
- 字段映射支持智能推荐（基于关键词模糊匹配）
- 查询页提供常用示例按钮，一键填充
- 错误提示使用中文，对常见错误提供解决建议

### 6.2 可维护性
- 模块化设计：`modules/` 下每个模块职责单一
- 后端路由与业务逻辑分离：`app.py` 仅做路由编排，业务逻辑委托给模块
- 配置集中管理：`config.py`

### 6.3 可扩展性
- 新增数据源：扩展 `DataProcessor` 支持新文件格式
- 新增完整性测试：在 `IntegrityChecker` 中添加新方法
- 切换 AI 模型：替换 `AICodeGenerator` 中的 API 端点和模型名

## 8. 边界与限制

### 8.1 当前版本不支持的场景
- 不支持多用户并发（Flask 开发服务器为单线程）
- 不支持跨会话数据持久化（无数据库）
- 不支持图表可视化
- 不支持多文件联合查询（但支持序时账+科目余额表双表完整性测试）

### 8.2 已知限制
- AI 代码质量依赖 DeepSeek 模型能力，复杂查询可能需要手动调整代码
- 超大文件（100 万行以上）的完整性测试可能超时
- 不含科目余额表时，测试二和测试三自动跳过

| DA数据清洗业务AI应用 | 自然语言驱动、本地安全、零代码 | 功能聚焦查询分析、不包含可视化 |

## 10. 路线图

### v1.3.0（当前）
- 单文件序时账导入 + 字段映射
- DeepSeek AI 代码生成 + 沙箱执行
- 三项完整性测试
- 科目余额表导入 + 交叉验证
- 财务报表清洗（AI 读前 10 行识别结构 + 规则提取）
- 科目余额表 ↔ 报表核对（规则映射 + 可编辑 + AI 差异分析）
- 多报表导入合并核对
- 结果导出 Excel/CSV

### 未来可能的迭代方向
- 图表可视化
- 多数据源联合查询
- 查询历史与模板保存
- 自定义完整性测试规则
- 更多 AI 模型支持（通义千问、GLM 等）
