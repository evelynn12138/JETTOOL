## 1. 快速开始

### 1.1 启动应用

```bash
cd DA数据清洗业务AI应用
python app.py
```

浏览器打开 `http://127.0.0.1:5003` 即可使用。

### 1.2 整体流程

```
上传文件 → 字段映射 → 完整性测试（可选）→ 查询分析
  步骤1       步骤2         步骤3            步骤4
                          ├─ 财务报表清洗（可选）
                          └─ 科目余额表核对（可选）
```

完整性测试、财务报表清洗、科目余额表核对均为可选步骤，可直接跳过进入查询分析。

### 1.3 AI 配置说明

系统默认使用 Dify Workflow API。如果你有 DeepSeek API Key，可在终端设置环境变量启用：

```bash
export DEEPSEEK_API_KEY="sk-你的密钥"
python app.py
```

## 2. 第一步：上传文件

### 2.1 支持的文件

| 格式 | 说明 |
|------|------|
| `.xlsx` | Excel 工作簿，支持多 Sheet |
| `.csv` | 文本格式，自动检测编码（UTF-8/GBK） |

### 2.2 上传序时账

1. 点击上传区域或拖拽文件到虚线框
2. 选择 Sheet 页（多 Sheet 时）和表头行
3. 点击「上传并解析」

> **提示**：如果 CSV 文件包含中文且出现乱码，系统会自动尝试 GBK 编码。

### 2.3 上传科目余额表（可选）

操作同上，用于完整性测试的交叉验证。

### 2.4 最大文件限制

单文件最大 4GB。

## 3. 第二步：字段映射

### 3.1 映射方式

系统提供三种方式将原始数据列映射到标准字段：

#### 方式一：AI 智能映射（推荐）

点击「AI 智能映射」按钮，AI 根据列名、数据类型和样本值自动推荐映射。

#### 方式二：手动选择

从下拉框选择源文件的列名，映射到对应的标准字段。

#### 方式三：手动输入常量值

对于「公司名」「币种」等字段，如果源文件没有对应列，可以手动输入一个固定值（所有行共用）。选择空选项后会出现输入框。

### 3.2 标准字段说明

#### 序时账标准字段

| 字段名 | 必填 | 说明 |
|--------|------|------|
| 日期 | ✅ | 交易发生日期 |
| 摘要 | ✅ | 交易内容 |
| 科目编号 | ✅ | 会计科目编号（将自动转为文本格式） |
| 科目名称 | ✅ | 会计科目名称 |
| 金额 | 视情况 | 交易发生额（有借方+贷方时可自动计算） |
| 凭证号 | ✅ | 凭证编号 |
| 公司名 | ✅ | 公司名称，可手动输入 |
| 借方 | 否 | 借方金额 |
| 贷方 | 否 | 贷方金额 |
| 方向 | 否 | 借贷方向，影响完整性测试 |
| 部门 | 否 | 所属部门 |
| 制单人 | 否 | 制单人员 |
| 币种 | 否 | 交易币种，可手动输入 |

#### 科目余额表标准字段

| 字段名 | 必填 | 说明 |
|--------|------|------|
| 公司名 | ✅ | 公司名称 |
| 科目编号 | ✅ | 会计科目编号 |
| 科目名称 | ✅ | 会计科目名称 |
| 期初余额 | ✅ | 期初余额（可选模式：借贷方计算） |
| 期末余额 | ✅ | 期末余额（可选模式：借贷方计算） |
| 币种 | 否 | 币种（多币种筛选用） |

### 3.3 科目余额表格式

根据源文件格式选择：

- **已有期初/期末余额**：直接映射期初余额、期末余额列
- **借贷方计算**：映射期初借方/贷方、期末借方/贷方，系统自动计算余额

### 3.4 金额处理方式

保存映射后，系统自动检测金额处理方式，在完整性测试中展示：

| 检测条件 | 处理方式 |
|----------|----------|
| 映射了「金额」字段 | 直接使用金额列 |
| 映射了「方向」+「金额」 | 借正贷负（借方为正、贷方为负） |
| 映射了「借方」+「贷方」 | 发生额 = 借方 - 贷方 |

> **提示**：导入时所有数值列的空值会自动替换为 0。

## 4. 第三步：完整性测试

完整性测试页面采用 **4 步引导流程**，按顺序操作即可。

### 概览：智能建议

页面顶部会显示蓝色建议卡片，系统根据数据分析给出的处理建议（无需手动配置）：

| 建议 | 含义 |
|------|------|
| 🔵 建议 | 高置信度，建议采纳 |
| 🟣 提示 | 中等置信度，可参考 |
| ⚪ 参考 | 低置信度，按需选择 |

### Step 1：预处理

#### 向下填充缺失字段

某些 ERP 导出的序时账只会在凭证首行填写凭证号和日期，后续行留空。

勾选需要填充的字段后，系统先按凭证分组填充，再做后续剔除，避免误删有效数据。

| 字段 | 建议 |
|------|------|
| 凭证号 | 推荐开启（空值 > 5% 时会提示） |
| 日期 | 推荐开启 |
| 制单人 | 按需开启 |

#### 数据清洗规则

| 规则 | 说明 |
|------|------|
| 剔除凭证编号为空的行 | 通常为合计行或空行 |
| 剔除科目余额表含"合计"的行 | 排除汇总统计行 |

#### 多币种筛选

如果科目余额表映射了币种字段且数据包含多种币种，可在此选择保留单一币种进行核对。

### Step 2：金额处理

系统自动识别金额处理方式，并显示确认卡片。

如果映射了「方向」字段，可启用**借正贷负方向调整**——借方金额为正、贷方金额为负。

### Step 3：特殊调整

#### 反结转（剔除损益结转凭证）

适用于金蝶、用友等系统：

1. 勾选「启用反结转」
2. 填写结转科目编号（默认 4103），不确定请留空
3. 填写摘要关键词（默认「结转, 损益」），摘要同时包含所有关键词才视为结转凭证

#### 仅用末级科目

如果科目余额表有父子科目层级，开启后只保留最细粒度的明细科目。

### Step 4：运行检查

#### 配置确认

进入第四步后，系统实时显示当前配置状态：

| 卡片 | 内容 |
|------|------|
| 预处理 | 启用的清洗规则数量 |
| 金额处理 | 处理模式 + 方向调整状态 |
| 特殊调整 | 反结转 + 末级科目启用状态 |

#### 运行三项检查

点击「运行全部三项检查」，系统依次执行：

| 检查项 | 说明 |
|--------|------|
| ① 序时账完整性 | 按凭证分组检查借贷是否平衡 |
| ② 科目余额表完整性 | 期初+发生额=期末，检查归零 |
| ③ 交叉验证 | 按公司+科目编号比对两表发生额 |

#### 结果操作

- **AI 差异分析**：让 AI 分析差异原因
- **导出为 Excel**：导出完整测试报告（含三项测试明细）
- **跳过，直接进入查询**：不保存结果，直接去查询分析

## 5. 财务报表清洗

位于完整性测试页面下方，用于处理资产负债表和利润表。

### 操作步骤

1. **上传文件**：拖拽或选择 `.xlsx` / `.xls` 文件
2. **选择 Sheet**：选择对应的工作表
3. **AI 识别结构**：系统读取前 10 行，自动判断报表类型、列映射、布局方式
4. **提取数据**：确认 AI 识别结果后，规则引擎自动清洗输出
5. **导入多报表**：可分别导入资产负债表和利润表合并核对

### 自动过滤的行

- 空行
- 合计行、小计行、总计行
- 签名行（单位负责人、制表人等）
- 注释行（"注："开头）
- 大类标题行（"流动资产："等）

### 输出格式

| 报表类型 | 输出列 |
|----------|--------|
| 资产负债表 | 项目名称、年初余额、期末余额 |
| 利润表 | 项目名称、期末余额（自动映射本年累计） |

## 6. 科目余额表核对

将清洗后的财务报表与科目余额表做逐项比对。

### 操作步骤

1. 导入一张或多张报表后，点击「核对科目余额表」
2. 左侧为科目余额表（每条科目记录）、右侧为核对结果
3. 查看差异，手动修正映射，点击「刷新核对」重新计算

### 映射方式

| 方式 | 说明 |
|------|------|
| 规则引擎自动映射 | 编号前缀匹配 + 名称关键词匹配（100+ 条规则） |
| AI 兜底 | 未匹配的科目由 AI 辅助推断 |
| 手动编辑 | 直接在左侧修改报表科目名称 |

### 一键优化

当差异较多时，点击「一键优化」：

1. 系统尝试所有合理的科目归属方案
2. 选择使总差异最小的方案
3. 展示调整明细，可选择「应用优化方案」或「放弃」

> **注意**：已匹配一致的项目会被锁定，不会被优化算法移动。

### 差异分析

- **AI 差异分析**：分析差异模式（映射调换、归属错误等）
- **导出结果**：导出为 Excel，包含科目余额表明细和核对结果两个 Sheet，新增「科目类型（参考）」列

## 7. 第四步：查询分析

### 7.1 三种查询方式

#### 方式一：预设筛选规则

页面左侧有三个规则包：

| 规则包 | 内容 |
|--------|------|
| 通用筛选规则 | 整数金额检测、大额交易、负金额检测、节假日交易等 7 条 |
| 行业常见异常凭证特征 | 频繁整数交易、月末集中交易、关联方交易 3 条 |
| EY GAM 异常凭证筛选 | 人工凭证检测、非常规科目配对、重大估计调整 3 条 |

选择规则 → 填写参数 → 点击「应用」→ 自动填入查询框。

#### 方式二：自然语言查询

直接在输入框用中文描述需求：

```
查询各个科目的总金额
筛选出管理费用2024年的借方发生额
按部门汇总并计算占比
```

#### 方式三：自行输入 + 优化表达

输入后点击「优化表达」，AI 会把模糊描述优化成更具体的表达。

### 7.2 查询流程

```
输入查询 → 优化表达（可选）→ 生成并执行 → 查看结果
                                              ├─ 编辑 SQL（可选）
                                              ├─ AI 自动复核
                                              ├─ 收藏查询
                                              └─ 导出结果
```

### 7.3 SQL 编辑器

点击「编辑 SQL」可进入 CodeMirror 编辑器修改生成的 SQL，修改后点击「执行」重新运行。

### 7.4 AI 自动复核

代码生成后自动触发四维复核：

| 维度 | 说明 |
|------|------|
| 语法检查 | DuckDB SQL 语法是否合规 |
| 安全审查 | 是否仅为 SELECT 只读操作 |
| 意图匹配 | SQL 是否准确反映了查询需求 |
| 性能优化 | 是否有明显的性能问题 |

### 7.5 收藏与导出

- **收藏**：将当前查询语句保存，方便后续复用
- **导出 CSV / Excel**：将查询结果下载到本地

## 8. 常见问题

### 8.1 上传 CSV 报编码错误怎么办？

系统会自动尝试 UTF-8 → GBK → GB2312 → Latin1 等多种编码。如果仍失败，建议在 Excel 中另存为 UTF-8 编码的 CSV。

### 8.2 科目编号显示为 1001.0 怎么办？

系统在导入时会自动将科目编号转为文本格式，不会出现 `.0` 后缀。如果之前的版本已导入，需要重新上传。

### 8.3 完整性测试报"科目余额表缺少: 科目编号"？

通常是因为 Excel 列名有尾部空格（如「账户 」而不是「账户」），导致字段映射匹配不上。v1.3.0 已自动处理列名空格，重新上传即可。

### 8.4 交叉验证结果不准确？

v1.3.0 已将交叉验证分组从「公司+科目编号+科目名称」改为「公司+科目编号」汇总，排除了科目名称不一致导致的干扰。

### 8.5 如何开启 AI 功能？

如果使用 Dify（默认配置），确保 Dify 服务可访问。如果使用 DeepSeek，设置环境变量：

```bash
export DEEPSEEK_API_KEY="sk-你的密钥"
python app.py
```

### 8.6 数据安全吗？

所有数据在本地机器处理。AI 调用时仅传输字段名和查询文本，原始财务数据不出本机。API Key 仅存于会话，不持久化。

## 9. 附录：术语说明

| 术语 | 说明 |
|------|------|
| **序时账** | 按时间顺序记录的凭证明细（总账流水） |
| **科目余额表** | 各会计科目的期初余额、本期发生额和期末余额汇总 |
| **完整性测试** | 检查数据是否满足会计基本恒等关系的三项测试 |
| **反结转** | 剔除损益结转凭证（期末将损益类科目余额结转至本年利润） |
| **末级科目** | 最细粒度的明细科目，无下级子科目 |
| **报表核对** | 将科目余额表与财务报表做逐项比对 |
| **借正贷负** | 借方金额取正、贷方金额取负的计算方式 |
| **交叉验证** | 序时账与科目余额表两表发生额的比对验证 |
