## 第二幕 · 掰开来看：问题出在哪

| 环节 | 内容 |
|------|------|
| 画面 | 屏幕上出现三个关键词，依次放大弹出。 |
| 音效 | 沉稳、有节奏的背景音乐渐入。 |

**关键词一：流程割裂**

| 环节 | 内容 |
|------|------|
| 画面 | 流程图：NAS 取数 → Excel 初步查看 → Alteryx 做完整性验证 → Alteryx 抽凭 → Teams沟通需求 → 再开一个 Alteryx 工作流 → 导出 → Excel 整理 → 发文件。箭头错综复杂。 |
| 旁白 | 我们自己的数据处理链路，横跨 NAS、Excel、Alteryx、Teams——至少三四个工具。 |
| 旁白 | 每切换一次，都是上下文的丢失和时间的消耗。而且所有操作都靠我们手动完成。 |

**关键词二：简单需求占用了大量时间**

| 环节 | 内容 |
|------|------|
| 画面 | 聊天记录快速滚动：按科目筛序时账、按金额区间抽凭、按部门汇总、TOD 抽样……翻了一页又一页。 |
| 旁白 | 仔细看看我们每天在处理的需求，本质上就这几类。 |
| 旁白 | 每一个单独看都不难，Alteryx 工作流已经做过很多次了。但架不住量大——一个项目几十个科目，全部门几十个项目同时跑，每个需求都要我们亲手打开 Alteryx、配参数、跑一次、导出来、发出去。 |
| 旁白 | 我们最有价值的能力——复杂数据分析、自动化工具开发、风险建模——反而被这些重复劳动挤占了时间。 |

**关键词三：沟通成本高**

| 环节 | 内容 |
|------|------|
| 画面 | 分屏：左边是Teams对话框（审计员发需求 → 等待），右边是 Alteryx 多工作流排队。 |
| 旁白 | 一个需求从发来到交付，真正"干活"的时间可能不到 10 分钟。 |
| 旁白 | 但加上沟通需求细节、等待审计员确认、文件传输、后续调整——一个来回少则两三个小时。 |
| 旁白 | 而且我们手头同时处理的请求不止一个，每个都在排队。 |

## 第四幕 · 财智灵契：TRDA 的加速器

| 环节 | 内容 |
|------|------|
| 画面 | 暖色调渐入。产品界面——浏览器中的工具首页。 |
| 音效 | 音乐转为明朗、有信心的调性。 |

| 环节 | 内容 |
|------|------|
| 画面 | 快速展示：上传序时账 + 科目余额表 → 确认字段映射 → 一键跑完三项完整性测试（结果直接展示）→ 输入一句中文查询"筛选管理费用借方大于50万的交易"→ AI 自动生成代码 → 执行 → 结果表格展示。全程约 1 分钟。 |
| 旁白 | 现在，同样的事情只需要一个浏览器界面。 |

| 环节 | 内容 |
|------|------|
| 画面 | 产品核心能力逐条展示，配对应界面截图。 |
| 旁白 | **第一，双表导入 + 智能字段映射。** 同时上传序时账和科目余额表，系统自动识别字段、推荐映射。不管被审计单位的数据格式长什么样，我们只需要确认一次，不用在 Alteryx 里手动配字段。 |
| 旁白 | **第二，一键完整性测试。** 三项固化审计程序——凭证借贷平衡检查、科目发生额归零验证、两表交叉比对——一键全部跑完，差异明细直接可导出为底稿附件。不用再在 Alteryx 里搭建工作流。 |
| 旁白 | **第三，自然语言查询 + AI 代码生成。** 用中文说出需求——"管理费用2024年借方发生额""按科目汇总并计算占比"——AI 实时生成 Pandas 代码并执行，结果即时展示。很多以前需要开 Alteryx 工作流的需求，现在一句话就解决了。 |
| 旁白 | **第四，代码可编辑。** 我们 TRDA 组有编程基础的同事，可以直接在 CodeMirror 编辑器里修改生成的代码——做更复杂的衍生计算、调聚合逻辑、加自定义条件。AI 做 80%，我们的专业能力做剩下的 20%。 |

| 环节 | 内容 |
|------|------|
| 画面 | 分屏对比：左边是传统 TRDA 工作流（NAS → Alteryx → Excel → 微信→ 来回调整），右边是财智灵契单一界面。 |
| 旁白 | 左边：NAS 取数 → Alteryx 配工作流跑完整性 → 导出发给审计组 → 审计组提需求 → Alteryx 再配工作流跑筛选 → 导出 → 传文件 → 调整再来。多个工具、多次沟通、多次操作。 |
| 旁白 | 右边：上传数据 → 确认映射 → 跑测试 → 查数据。一个工具，一次完成。 |

| 环节 | 内容 |
|------|------|
| 画面 | 重点展示：同一个界面内完成序时账 + 科目余额表完整性测试 + 抽凭分析的全流程。 |
| 旁白 | 财智灵契不是替代 Alteryx——它是把我们在 Alteryx、Excel、Teams之间反复切换的环节，整合到一个界面里，把我们从重复性操作中解放出来。 |

| 环节 | 内容 |

| 旁白 | 对审计组来说，意味着他们不再需要等排期。想筛什么数据，自己打开浏览器，一句话搞定。 |

## 附：脚本使用说明

- **总时长估算**: 约 4-5 分钟
- **关键叙事视角**: 全程以 TRDA 组成员的第一人称展开。"我们 TRDA 组"是叙事主体，"审计员/审计组"是服务对象
- **核心信息层次**:
  - 第一层：TRDA 组的工作量来自大量重复性请求
  - 第二层：工具让 TRDA 组自己处理请求更快
  - 第三层（更关键）：工具交给审计组自服务，从根本上减少 TRDA 的请求量
