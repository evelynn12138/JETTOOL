# 维护说明

## 1. Dify AI 代理配置

### 主要配置（config.py）

所有 Dify API 配置集中在 `config.py` 中，共两套：

| 用途 | 配置变量 | 说明 |
|------|---------|------|
| SQL生成、字段映射、报表清洗等 | `DIFY_MAIN_BASE_URL` + `DIFY_MAIN_API_KEY` | 主 Dify Workflow（Qwen3-235B） |
| SQL 代码复核审查 | `DIFY_REVIEW_BASE_URL` + `DIFY_REVIEW_API_KEY` | 复核 Dify Workflow（qwen3-30B） |

### 修改方法

编辑 `config.py`，找到以下位置：

```python
# 主要 Dify Workflow — 用于 SQL 生成、字段映射、报表清洗、AI 差异分析等
DIFY_MAIN_BASE_URL = "https://ai-platform-uat.ey.net/v1"
DIFY_MAIN_API_KEY = "app-ARr9EIyUv4i5RXPNW2CwdoP5"
# 复核 Dify Workflow — 用于 SQL 代码复核审查
DIFY_REVIEW_BASE_URL = "https://ai-platform-uat.ey.net/v1"
DIFY_REVIEW_API_KEY = "app-uuvpkgtY94HcQLgxDmaR5QLi"
```

需要修改时：
- **更换地址**：改 `DIFY_MAIN_BASE_URL` 或 `DIFY_REVIEW_BASE_URL`
- **更换 API Key**：改 `DIFY_MAIN_API_KEY` 或 `DIFY_REVIEW_API_KEY`
- **更换模型**：需要在 Dify Workflow 端修改，无需在代码中变更

### 获取新 API Key

1. 登录 Dify 平台工作流页面
2. 进入「API 访问」页
3. 复制以 `app-` 开头的 API Key

## 2. 文件上传限制

文件大小上限在 `config.py` 中：

```python
MAX_CONTENT_LENGTH = 4096 * 1024 * 1024  # 4GB
```

按需调整数字即可（单位：字节）。

## 3. 预设筛选规则

规则存储在 `temp/preset_rules.json`，JSON 格式。可直接编辑该文件增删规则。每条规则结构：

```json
{
  "id": "rule_id",
  "name": "规则名称",
  "description": "规则描述",
  "category": "分类",
  "sql_template": "SELECT * FROM \"data\" WHERE ...",
  "params": [
    {"name": "param_name", "label": "参数标签", "type": "text"}
  ]
}
```

## 4. PyInstaller 打包

运行 `electron/build.js` 或直接用 PyInstaller：

```bash
pyinstaller --onefile --add-data "templates:templates" --add-data "static:static" run.py
```

打包后的应用数据目录在 `~/.da-cleaner/`。
