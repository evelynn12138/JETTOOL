# DA数据清洗业务AI应用 —— 产品介绍

## 审计数据处理，为什么一直这么麻烦？

先描述一个你在项目上一定见过的场景。

被审计单位的序时账和科目余额表从 NAS 公盘拿到以后，要先发给 TRDA 组的同事。TRDA 组打开 Alteryx 做完整性验证：凭证借贷是否平衡、科目发生额是否归零、两表数据能不能对上。验证通过，再按审计员的要求做抽凭、按科目筛选序时账、TOD 抽样。一来一回，少则半天，多则一两天。

更常见的情况是：审计员在项目上需要一份某个科目的序时账明细——需求本身不复杂，"帮我筛一下管理费用今年全年的数据"——但这句话意味着你要去联系 TRDA 组的同事，等人家的排期，等人家打开 Alteryx 跑一遍流程，再把文件发回来。每一次这样的请求，都是一次跨团队沟通加排队等待。

这个链路有什么问题？三个。

**第一，流程割裂。** 整个数据处理链路横跨 NAS 文件共享、Excel、Alteryx、Teams聊天、审计底稿——至少四五个工具和环境。每切换一次，都是一次上下文的丢失和时间的消耗。

**第二，低效协作。** TRDA 组是集中服务团队，面对的是全部门的需求。审计员要一份序时账筛选，排队等 TRDA；TRDA 打开 Alteryx 跑一次，返回结果；审计员发现还需要调整条件，再次排队。需求本身不复杂，但沟通成本和等待时间占了整个环节的大头。

**第三，简单需求重复劳动。** 很多审计员发来的需求，本质上就这几类：按科目筛选序时账、按金额区间抽凭、按部门汇总发生额、TOD 抽样。每一个单独看都不难，但架不住量大——一个项目几十个科目，每个都来一遍，TRDA 组的工作量非常可观。

算一笔账：一次简单的"帮我筛一下某某科目的数据"，从发消息到拿到结果，平均需要 2 到 4 个小时——其中真正"干活"的时间不到 10 分钟。剩下的时间全部花在沟通、排队等待、文件传输上。一个中型项目，这样的请求少则十几次，多则几十次。

核心问题不是工具不够强，而是工具不属于真正需要用它的人。Alteryx 很强，但它不是每个审计员桌面上的工具。TRDA 组很专业，但每接一次单都是对双方时间的消耗。

## DA数据清洗业务AI应用：一站式解决

DA数据清洗业务AI应用做的事情很简单：把分散在 NAS、Excel、Alteryx、Teams沟通中的数据处理环节，整合到一个浏览器界面里。

**第一步，双表导入与字段映射。** 同时上传序时账和科目余额表，系统自动识别字段、推荐映射。不管被审计单位的数据格式长什么样，你只需要确认一次。

**第二步，一键完整性测试。** 系统内置三项固化审计程序：凭证借贷平衡检查、科目发生额归零验证、序时账与科目余额表交叉比对。全部一键运行，差异明细直接可导出为底稿附件。

**第三步，自然语言查询与 AI 代码生成。** 用中文说出你的需求——"筛选管理费用2024年全年的借方发生额"、"按科目汇总并计算占比"——AI 实时生成 Pandas 代码并执行，结果即时展示。

**第四步，代码可编辑。** 有 Python 基础的同事可以直接在 CodeMirror 编辑器中修改生成的代码，做更灵活、更个性化的数据分析，不受预置功能限制。AI 负责完成 80% 的基础工作，剩下的 20% 交给你的专业判断。

如果生成的代码第一次执行失败，系统会自动判断错误类型——空值问题、类型不匹配、格式不一致——然后自动增强清洗后重试，最多尝试三次。这不是一个"要么出结果、要么报错"的系统，而是一个会自己想办法解决问题的系统。

安全方面，AI 生成的代码经过三层沙箱防护：AST 静态分析检查代码结构、受限执行环境禁止敏感操作、30 秒超时自动终止。数据始终在本地，唯一联网的是向 DeepSeek API 发送查询文本。API Key 仅存于会话，不持久化、不入库、不记日志。

## 给审计团队一个新的工作方式

对比一下传统链路和DA数据清洗业务AI应用：

传统链路：NAS 取数 → Excel 预览 → TRDA 做完整性测试 → Alteryx 筛选 → Teams传文件 → Excel 整理。多工具、多人、多环节，一次请求 2 到 4 小时。

DA数据清洗业务AI应用：上传 → 确认映射 → 查询 → 出结果。一个人，一个工具，几分钟。

这带来几个实质性的改变：

**一站式。** 序时账导入、完整性测试、科目余额表核验、查询分析、抽凭，全在一个界面。不再需要在 NAS、Excel、Alteryx 之间来回切换。

**自服务。** 审计员可以自己完成大部分常规数据查询——筛科目、做汇总、按条件筛选——不用每次都给 TRDA 组发消息排队等。TRDA 组可以更聚焦在真正复杂的数据任务上。

**零门槛 + 可扩展。** 不会代码的同事说中文就能查。会代码的同事可以进一步编辑和优化生成的代码，两者互不冲突。

**零成本启动。** 一台笔记本电脑，一个每天只需要几分钱的 DeepSeek API Key。不需要部署服务器，不需要采购许可证，不需要申请 IT 资源。一行命令 `python app.py` 就能跑起来。

DA数据清洗业务AI应用不是一个颠覆性的产品。它不做你做不到的事——它只是把审计项目中分散在多个工具、多个环节、多个人员之间的数据处理工作，整合到一个界面里。让数据服务于审计判断，而不是反过来。
